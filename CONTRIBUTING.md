# Come contribuire

Grazie per voler contribuire ad **awesome-api-italia**! Questa lista esiste grazie alla community.

## Criteri di inclusione

Una voce può essere aggiunta se rispetta **tutti** i seguenti requisiti:

- Espone dati italiani **oppure** è erogata da un ente o società italiana
- Ha documentazione accessibile (anche solo in italiano)
- È attivamente mantenuta (ultimo aggiornamento entro 2 anni, salvo eccezioni per risorse istituzionali stabili)
- Non è già presente nella lista

## Formato delle voci

Ogni voce segue questo formato:

```
- [emoji(s)] [Nome](URL) — Descrizione concisa in italiano. Dettaglio tecnico o differenziante.
```

**Emoji da usare:**
- 💰 — A pagamento o con piano freemium limitato
- 🏛️ — Ente pubblico / PA
- 🔑 — Richiede registrazione o API key
- 📄 — Documentazione ufficiale disponibile

Le emoji si combinano. Esempio: `🏛️ 🔑` per una PA che richiede registrazione.

## Come aprire una PR

1. Fai il fork del repository
2. Aggiungi la voce nella sezione appropriata, in ordine alfabetico o per rilevanza
3. Verifica che il link funzioni e la documentazione sia accessibile
4. Apri una Pull Request con titolo descrittivo (es. `feat: add Cerved API in Rating section`)
5. Nella descrizione della PR spiega brevemente perché l'aggiunta è rilevante

## Voci deprecate

Le API non più manutenute, ritirate o con documentazione inaccessibile vanno spostate in [`DEPRECATED.md`](DEPRECATED.md) anziché essere eliminate, per mantenere traccia storica.

## Stile

- Descrizioni in **italiano**
- Niente punto finale nella descrizione
- Massimo 2 frasi per voce
- Non usare espressioni promozionali ("il migliore", "il più potente", ecc.)
- Le sezioni seguono l'ordine definito nel sommario del README

## Segnalare problemi

Apri una [Issue](https://github.com/openapi/awesome-api-italia/issues) per:
- Link non funzionanti
- API deprecate da spostare
- Nuove sezioni o riorganizzazioni
